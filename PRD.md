# PRD: Markdown 转公众号排版工具

## 1. 概述

### 1.1 产品名称
MD2WeChat — Markdown 转公众号排版工具

### 1.2 产品定位
一个纯前端的 Markdown 到微信公众号富文本排版工具，支持实时预览、多主题切换、样式微调，一键复制后可直接粘贴到公众号编辑器。

### 1.3 目标用户
内部团队成员（编辑、运营、技术人员）

### 1.4 核心价值
- **效率**：Markdown 写作 → 一键转换 → 直接粘贴到公众号，省去手动排版
- **一致性**：预设主题保证团队公众号文章风格统一
- **零门槛**：单 HTML 文件，无需安装任何依赖

## 3. 非功能需求

### 3.1 技术约束

| 约束 | 说明 |
|------|------|
| 单文件 | 整个应用为单个 HTML 文件，CSS/JS 全部内联 |
| 零依赖 | 不引入任何外部 CDN、JS 库（Markdown 解析自行实现或内联轻量库） |
| 存储 | 全部使用 localStorage，无后端 |

### 3.2 Markdown 解析策略

由于"零外部依赖"约束，Markdown 解析有两个方案：

- **方案 A**：内联一个轻量 Markdown 解析库（如 marked.js minified，约 30KB）直接嵌入 HTML
- **方案 B**：自行实现简化版 Markdown 解析器

**推荐方案 A** — 内联 marked.js。理由：覆盖语法全面、稳定性好、单文件内联不算外部依赖。

### 3.3 浏览器兼容

- 目标：Chrome 最新版（内部团队使用，不做宽泛兼容）
- 复制功能依赖 Clipboard API，需 HTTPS 或 localhost

### 3.4 部署

- **Phase 1**：产出单 HTML 文件，双击可用
- **Phase 2**：通过 GitHub Pages 部署到线上

## 8. 风险与决策记录

| 风险 | 影响 | 应对 |
|------|------|------|
| Markdown 解析复杂度 | 自实现可能语法覆盖不全 | 推荐内联 marked.js |
| 公众号样式兼容 | 公众号编辑器会过滤部分 CSS | 使用内联 style 而非 class |
| 图片外链限制 | 公众号不支持外链图片 | 本工具仅做预览，用户需自行上传图片到公众号 |

**关键决策：**
- 公众号渲染必须使用**内联 style**（不能用 class），因为公众号编辑器会剥离 `<style>` 标签和 class
- 代码块使用带背景色的 `<section>` + `<pre>` 样式块，不做语法高亮
